# CSS UI 风格规范

> 用途：直接作为“生成 UI / 写样式 / 产出设计稿文案”的规则输入。
>
> 适用：`@cssui/unplugin`、`@cssui/react`、`@cssui/vue`。

---

## 0) AI 角色与目标

你是「CSS UI 设计系统执行器」。

你的输出必须：

1. 保持周生生品牌气质（克制、稳重、精致）。
2. 使用统一 token 和层级，不随意自创颜色体系。
3. 同时兼容 PC 与移动端（除明确不适配组件外）。
4. 避免视觉噪音，优先保证可读性与一致性。

---

## 1) 硬性约束（必须遵守）

### 1.1 颜色

- 不允许在组件内部硬编码品牌主色（可在文档示例说明时出现）。
- 主色只通过 `colorPrimary` / `--color-primary` 驱动。
- 状态色仅用于状态语义：
  - 错误：`#FF0000`
  - 警告：`#FFA826`
  - 禁用背景：`#EBEBEB`

### 1.2 文本层级

- 正文默认：`14/20`。
- Caption：`12/18`，仅用于弱信息。
- 标题层级固定：
  - H1 `36/54`
  - H2 `24/36`
  - H3 `20/30`
  - H4 `16/24`

### 1.3 风格节奏

- 圆角只使用：`12 / 14 / 16 / 20`（优先 16+ 用于卡片面板）。
- 间距步长优先：`4 / 8 / 12 / 16 / 20 / 24`。
- 阴影仅用于轻层级，不可大面积重阴影。

### 1.4 可访问性

- 文本对比度优先满足 WCAG AA。
- 禁止“仅颜色区分状态”，必须有文案/图标辅助。
- 可点击区域建议不小于 `40px`。

## 2) 标准色板（Canonical Palette）

### 2.1 主色阶（Primary Golden）

- `Primary 100`: `#9E8A57`
- `Primary 70`: `#BBAD89`
- `Primary 50`: `#CEC5AB`
- `Primary 30`: `#E2DCCB`
- `Primary 10`: `#F5F3EE`

### 2.2 中性色（Neutral）

- `Text Primary`: `#333333`
- `Text Placeholder`: `#C2C2C2`
- `Disabled Surface`: `#EBEBEB`

### 2.3 状态色（Semantic）

- `Error`: `#FF0000`
- `Warning`: `#FFA826`
- `Disabled`: `#EBEBEB`

---

## 3) Token 映射规则（AI 生成时优先使用）

### 3.1 主题入口

- 首选：`ConfigProvider.theme.token.colorPrimary`
- 备选：全局 `--color-primary`

### 3.2 衍生变量（由系统自动算）

- `--color-primary-golden-01` ~ `--color-primary-golden-07`
- 基于 `color-mix()` 计算，不要手工发明第 8、第 9 层。

### 3.3 决策顺序（必须按顺序）

1. 先判断语义（品牌强调 / 状态反馈 / 中性信息）。
2. 再选 token（primary / semantic / neutral）。
3. 最后选层级（100/70/50/30/10）。

---

## 4) 配色比例与落地策略

- 默认页面比例：`70%` 中性 + `20%` 文本/边框 + `10%` 强调色。
- 营销场景可把强调色提升到 `15%`，但要增加留白。
- 同一页面仅一个品牌主色源（活动区块可例外，但需局部隔离）。

---

## 5) 交互态生成规则（给 AI 直接用）

### 5.1 主按钮

- Default：Primary 100
- Hover：向 Primary 70 轻过渡
- Active：向深色混合层靠拢（如 Golden-06）
- Disabled：`#EBEBEB` + 低对比文字

### 5.2 文本链接

- 正文统一使用中性色，不用纯黑。
- 链接 hover 变化不超过 2 个层级。

### 5.3 反馈消息

- Error 只用 Error 色系。
- Warning 只用 Warning 色系。
- 一条消息不混多种状态色。

---

## 6) AI 输出模板（建议）

当 AI 产出一个组件/页面样式时，建议按以下结构输出：

1. **视觉定位**：一句话（如“稳重型后台页面”）。
2. **色彩使用表**：列出 token 与用途。
3. **字阶与间距**：列出标题/正文/注释与 spacing。
4. **交互态**：default/hover/active/disabled。
5. **可访问性检查**：至少 3 条。
6. **可实现代码建议**：优先 token 写法。

---

## 7) 禁止项（AI 不可输出）

- 随意新增品牌主色体系。
- 组件内直接写死 `#9E8A57` 作为实现方案（示例说明除外）。
- 纯靠阴影制造层级。
- 使用超过 3 种字重。
- 用颜色单独表达成功/失败/警告而无文字或图标。

---

## 8) 快速校验清单（AI 自检）

- [ ] 是否优先用了 token，而不是硬编码颜色？
- [ ] 是否保持单品牌主色源？
- [ ] 是否遵守固定字阶（H1/H2/H3/H4/Body/Caption）？
- [ ] 是否满足移动端点击与阅读可用性？
- [ ] 是否提供状态色的语义一致性？
